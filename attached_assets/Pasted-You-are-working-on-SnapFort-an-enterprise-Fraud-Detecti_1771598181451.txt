All AI responses must be wrapped into a consistent envelope:

{

  "title": "string",
  "summary": "string",
  "confidence": 0.0,
  "entities": {
    "transaction_id": "string|null",
    "customer_id": "string|null",
    "case_id": "string|null"
  },
  "sections": [
    {
      "type": "risk_factors | table | bullets | timeline | chart | document | metrics | json",
      "heading": "string",
      "items": []
    }
  ],
  "actions": [
    { "type": "create_case | add_to_case | draft_sar | view_customer | open_graph | export", "label": "string", "payload": {} }
  ],
  "raw_text": "original model output"
}


If the model cannot return JSON reliably, implement a post-processor that converts plain text into this envelope.

2) INTENT DETECTION (MUST BE RELIABLE)

Implement intent detection using a hybrid approach:

A) Primary: Explicit Intent Tag (Preferred)

Update the AI system prompt to return:

INTENT: <intent_type>

Then output in JSON using the envelope above

B) Secondary: Heuristic Classifier (Fallback)

If intent missing or JSON invalid, detect intent based on user prompt + response content:

Rules (examples)

If mentions TXN- / “transaction” / “flagged” / “risk drivers” → transaction_analysis

If mentions CUST- / “customer behavior” / “peer group” → customer_analysis

If mentions “Draft SAR” / “SAR narrative” / “suspicious activity report” → sar_draft

If mentions “this week” / “trend” / “rate” / “top indicators” / “distribution” → analytics_summary

If mentions “case” / “timeline” / “escalated” / “assigned analyst” → case_summary

Else → general_answer

Return a confidence score:

0.9+ for clear matches

0.6–0.8 for partial

<0.6 triggers “general_answer” fallback renderer
